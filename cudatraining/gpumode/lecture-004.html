03 · TERMS용어와 맥락

이 강의를 읽기 위한 작은 사전.

Glossary11 entries
kernel — A GPU program launched over many threads/blocks to process data in parallel.
warp — A group of GPU threads that execute in lockstep on NVIDIA GPUs.
CUDA — NVIDIA's GPU programming platform and kernel programming model.
shared memory — Program-managed on-chip SRAM used for tiling and reuse within a block.
tiling — Splitting work/data into blocks that fit cache/shared memory/registers.
occupancy — How many warps/blocks can reside on an SM, constrained by registers, shared memory, and launch shape.
PyTorch — The deep learning framework often used here as the host/runtime integration layer.
fusion — Combining operations into fewer kernels to reduce memory traffic and launch overhead.
Triton — A Python-embedded DSL/compiler for writing high-performance GPU kernels.
Tensor Core — Matrix-multiply hardware units optimized for mixed-precision GEMM-like workloads.
scheduling — Choosing execution order, tiling, parallelism, and memory movement strategy.
“용어를 외우는 게 아니라, 어떤 병목을 가리키는 이름인지 붙잡는다.”study rule

For CUDA kernels, this lecture is relevant because it touches implementation details such as launch structure, memory movement, profiling, or low-level GPU execution. For Triton kernels, it helps connect Python-level kernel authoring to tiling, program IDs, compiler lowering, and autotuned performance. For ML compilers, it is useful for understanding how higher-level tensor programs become IR, PTX/SASS, scheduled kernels, or fused operators. For GPU architecture, the matmul/Tensor Core/CUTLASS/CuTe themes are the bridge between hardware matrix instructions and production kernels. For profiling and optimization, the lecture gives hooks for thinking about bandwidth, occupancy, tiling, benchmarking, and bottleneck diagnosis.
