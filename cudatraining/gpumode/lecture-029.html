03 · TERMS용어와 맥락

Glossary11 entries
Triton — A Python-embedded DSL/compiler for writing high-performance GPU kernels.
kernel — A GPU program launched over many threads/blocks to process data in parallel.
CUDA — NVIDIA's GPU programming platform and kernel programming model.
PTX — NVIDIA's virtual GPU ISA used as an intermediate target before SASS.
LLVM — Compiler infrastructure used in many lowering/code-generation stacks.
warp — A group of GPU threads that execute in lockstep on NVIDIA GPUs.
CTA — Cooperative Thread Array, commonly called a thread block in CUDA.
PyTorch — The deep learning framework often used here as the host/runtime integration layer.
SASS — NVIDIA machine code executed by the GPU.
MLIR — Compiler infrastructure for representing and transforming programs across dialects.
SM — Streaming Multiprocessor, the GPU execution unit that schedules warps.
“용어를 외우는 게 아니라, 어떤 병목을 가리키는 이름인지 붙잡는다.”study rule

For CUDA kernels, this lecture is relevant because it touches implementation details such as launch structure, memory movement, profiling, or low-level GPU execution. For Triton kernels, it helps connect Python-level kernel authoring to tiling, program IDs, compiler lowering, and autotuned performance. For ML compilers, it is useful for understanding how higher-level tensor programs become IR, PTX/SASS, scheduled kernels, or fused operators.
