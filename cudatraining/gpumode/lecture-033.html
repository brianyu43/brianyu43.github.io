03 · TERMS용어와 맥락

이 강의를 읽기 위한 작은 사전.

Glossary11 entries
kernel — A GPU program launched over many threads/blocks to process data in parallel.
MMA — Matrix multiply-accumulate instruction family used by Tensor Cores.
Triton — A Python-embedded DSL/compiler for writing high-performance GPU kernels.
scheduling — Choosing execution order, tiling, parallelism, and memory movement strategy.
reduction — Parallel accumulation pattern such as sum, max, min, or norm.
quantization — Representing values with fewer bits to reduce memory and compute cost.
CUDA — NVIDIA's GPU programming platform and kernel programming model.
shared memory — Program-managed on-chip SRAM used for tiling and reuse within a block.
PTX — NVIDIA's virtual GPU ISA used as an intermediate target before SASS.
scan — Prefix-sum/prefix-operation primitive used in parallel algorithms.
normalization — Operations such as LayerNorm/RMSNorm that rescale activations.
“용어를 외우는 게 아니라, 어떤 병목을 가리키는 이름인지 붙잡는다.”study rule

For CUDA kernels, this lecture is relevant because it touches implementation details such as launch structure, memory movement, profiling, or low-level GPU execution. For Triton kernels, it helps connect Python-level kernel authoring to tiling, program IDs, compiler lowering, and autotuned performance. For ML compilers, it is useful for understanding how higher-level tensor programs become IR, PTX/SASS, scheduled kernels, or fused operators. For GPU architecture, the matmul/Tensor Core/CUTLASS/CuTe themes are the bridge between hardware matrix instructions and production kernels.
