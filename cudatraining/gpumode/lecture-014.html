03 · TERMS용어와 맥락

이 강의를 읽기 위한 작은 사전.

Glossary8 entries
kernel — A GPU program launched over many threads/blocks to process data in parallel.
Triton — A Python-embedded DSL/compiler for writing high-performance GPU kernels.
CUDA — NVIDIA's GPU programming platform and kernel programming model.
benchmarking — Measuring performance carefully with warmup, synchronization, and representative sizes.
shared memory — Program-managed on-chip SRAM used for tiling and reuse within a block.
SM — Streaming Multiprocessor, the GPU execution unit that schedules warps.
PTX — NVIDIA's virtual GPU ISA used as an intermediate target before SASS.
NCU — Nsight Compute, used for kernel-level performance counters and analysis.
“용어를 외우는 게 아니라, 어떤 병목을 가리키는 이름인지 붙잡는다.”study rule

For CUDA kernels, this lecture is relevant because it touches implementation details such as launch structure, memory movement, profiling, or low-level GPU execution. For Triton kernels, it helps connect Python-level kernel authoring to tiling, program IDs, compiler lowering, and autotuned performance. For ML compilers, it is useful for understanding how higher-level tensor programs become IR, PTX/SASS, scheduled kernels, or fused operators. For profiling and optimization, the lecture gives hooks for thinking about bandwidth, occupancy, tiling, benchmarking, and bottleneck diagnosis.
