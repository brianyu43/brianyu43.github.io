03 · TERMS용어와 맥락

이 강의를 읽기 위한 작은 사전.

Glossary14 entries
PTX — NVIDIA's virtual GPU ISA used as an intermediate target before SASS.
kernel — A GPU program launched over many threads/blocks to process data in parallel.
warp — A group of GPU threads that execute in lockstep on NVIDIA GPUs.
CUDA — NVIDIA's GPU programming platform and kernel programming model.
SM — Streaming Multiprocessor, the GPU execution unit that schedules warps.
Triton — A Python-embedded DSL/compiler for writing high-performance GPU kernels.
occupancy — How many warps/blocks can reside on an SM, constrained by registers, shared memory, and launch shape.
TMA — Tensor Memory Accelerator, hardware support for bulk tensor movement on Hopper-class GPUs.
reduction — Parallel accumulation pattern such as sum, max, min, or norm.
NCU — Nsight Compute, used for kernel-level performance counters and analysis.
SASS — NVIDIA machine code executed by the GPU.
FlashAttention — An attention algorithm/kernel family that reduces HBM traffic with online softmax and tiling.
HBM — High Bandwidth Memory, the off-chip memory backing large tensor data.
fusion — Combining operations into fewer kernels to reduce memory traffic and launch overhead.
“용어를 외우는 게 아니라, 어떤 병목을 가리키는 이름인지 붙잡는다.”study rule

For CUDA kernels, this lecture is relevant because it touches implementation details such as launch structure, memory movement, profiling, or low-level GPU execution. For Triton kernels, it helps connect Python-level kernel authoring to tiling, program IDs, compiler lowering, and autotuned performance. For ML compilers, it is useful for understanding how higher-level tensor programs become IR, PTX/SASS, scheduled kernels, or fused operators. For LLM inference/training systems, the attention/KV-cache/softmax themes map directly to throughput, memory bandwidth, and serving latency. For profiling and optimization, the lecture gives hooks for thinking about bandwidth, occupancy, tiling, benchmarking, and bottleneck diagnosis.
